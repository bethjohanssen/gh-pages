命名规范

  1. 通用命名规则。
    1. 函数命名，变量命名，文件命名要有描述性；少用缩写。
    2. 在保持一个标识符意思明确的同时，应当尽量缩短其长度。
    3. 不要出现仅靠大小写区分的相似的标识符。
    4. 程序中不要出现名字完全相同的局部变量和全局变量，尽管两者的作用域不同而不会发生语法错误，但容易使人误解。~~~~完全没有遵守的规则😂
  2. 命名空间命名。 不使用缩写作为名称。
  3. 类型命名 。 类型名称的每个单词首字母均大写，不包含下划线： MyExcitingClass，MyExcitingEnum。 所有类型命名 —— 类, 结构体, 类型定义 (typedef)，枚举，类型模板参数 —— 均使用相同约定，即以大写字母开始，每个单词首字母均大写，不包含下划线。 类的命名推荐用”名词”或”形容词＋名词 “的形式。
  4. 函数命名。 常规函数使用大小写混合，取值和设值函数则要求与变量名匹配。 一般来说，函数名的每个单词首字母大写 (即 “驼峰变量名” 或 “帕斯卡变量名”)，没有下划线。对于首字母缩写的单词，更倾向于将它们视作一个单词进行首字母大写 (例如，写作 StartRpc() 而非 StartRPC())。 函数名应当使用”动词”或者”动词＋名词 “（动宾词组）的形式。 所有返回布尔的函数应发起true/false的询问，如IsVisible() 或 ShouldClearBuffer()。
  5. 变量命名 。 变量 (包括函数参数) 和数据成员名一律小写，单词之间用下划线连接。 类的成员变量以下划线结尾，但结构体的就不用，如： ~~`a_local_variable`， a_struct_data_member， a_class_data_member_。~~~~表示疑惑的规则。~~ 变量的名字应当使用“名词”或者“形容词＋名词 ”。
  6. 常量命名。 声明为 constexpr 或 const 的变量，或在程序运行期间其值始终保持不变的，命名时以 “k” 开头，大小写混合。 所有具有静态存储类型的变量 (例如静态变量或全局变量, 参见 存储类型) 都应当以此方式命名。对于其他存储类型的变量，如自动变量等，这条规则是可选的。 如果不采用这条规则， 就按照一般的变量命名规则。
  7. 枚举命名。 枚举的命名和常量一致: kEnumName 。
  8. 宏命名。 全部大写，使用下划线。

doxygen 注释风格

VS2019 生成的文档注释样式：Doxygen(/**)

  • brief description 简要说明。简短一行。
  • detailed description 详细描述。更长，更详细的文档。目前使用的是Javadoc风格。 ```C++ /**
  • … text …
  • / ```
  • body description 可以作为详细描述，也可以描述详细实现的集合。

在成员之后放置文档

如果要记录文件、结构、联合、类或枚举的成员，有时需要将文档块放在成员之后而不是之前。 为此，您必须在注释块中放置一个额外的 < 标记。 请注意，这也适用于函数的参数。

///< Detailed description after the member
/**< [in] docs for input parameter v. */
